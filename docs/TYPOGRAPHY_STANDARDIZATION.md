# 타이포그래피 사이즈 표준화 (CTO 리뷰)

**작성일**: 2026-02-12  
**목적**: 텍스트 사이즈 표준화로 확장성/일관성/유지보수성 향상

---

## 표준화 요구사항

- ✅ **26px**: 유지 (`--fs-2xl`)
- ✅ **12px, 13px** → **12px**로 통일 (`--fs-micro`)
- ✅ **15px, 16px** → **15px**로 통일 (`--fs-base`)
- ✅ **17px, 18px** → **18px**로 통일 (`--fs-xl`)

---

## 현재 상태 분석

### 변수 정의
```css
--fs-micro: 12px;  /* 유지 */
--fs-xs: 13px;     /* → --fs-micro로 통일 */
--fs-sm: 14px;     /* 유지 */
--fs-base: 15px;   /* 유지 */
--fs-md: 16px;     /* → --fs-base로 통일 */
--fs-lg: 17px;     /* → --fs-xl로 통일 */
--fs-xl: 18px;     /* 유지 */
--fs-2xl: 26px;    /* 유지 */
```

### 사용 현황

| 변수 | 크기 | 사용 빈도 | 변경 후 |
|------|------|----------|---------|
| `--fs-xs` | 13px | 2회 | `--fs-micro` (12px) |
| `--fs-md` | 16px | 5회 | `--fs-base` (15px) |
| `--fs-lg` | 17px | 3회 | `--fs-xl` (18px) |

---

## 변경 전략

### 옵션 1: 변수 제거 및 통일 (권장)

**핵심 원칙**: 중복 변수 제거, 표준 변수로 통일

```css
:root {
  /* === 타이포그래피 표준 스케일 (5단계) === */
  --fs-micro: 12px;    /* 12px, 13px 통일 */
  --fs-sm: 14px;      /* 유지 */
  --fs-base: 15px;    /* 15px, 16px 통일 */
  --fs-xl: 18px;      /* 17px, 18px 통일 */
  --fs-2xl: 26px;     /* 유지 */
  
  /* === 레거시 호환성 (선택사항) === */
  /* 기존 코드 호환을 위해 일시적으로 유지 가능 */
  --fs-xs: var(--fs-micro);  /* 13px → 12px */
  --fs-md: var(--fs-base);   /* 16px → 15px */
  --fs-lg: var(--fs-xl);     /* 17px → 18px */
}
```

**장점**:
- ✅ 변수 개수 감소: 8개 → 5개 (37.5% 감소)
- ✅ 명확한 구조: 중복 제거
- ✅ 확장성: 새로운 크기 추가 시 명확한 위치 결정
- ✅ 레거시 호환성: 기존 코드와 호환

**단점**:
- ⚠️ 시각적 변경: 크기 변경으로 인한 레이아웃 영향 (최소화)

---

## 적용 계획

### Phase 1: 변수 값 변경 (즉시 적용)

1. `--fs-xs: 13px` → `--fs-xs: var(--fs-micro)` (12px)
2. `--fs-md: 16px` → `--fs-md: var(--fs-base)` (15px)
3. `--fs-lg: 17px` → `--fs-lg: var(--fs-xl)` (18px)

**영향 범위**: 
- `--fs-xs` 사용처: 2곳 (시각적 변경: 13px → 12px)
- `--fs-md` 사용처: 5곳 (시각적 변경: 16px → 15px)
- `--fs-lg` 사용처: 3곳 (시각적 변경: 17px → 18px)

### Phase 2: 사용처 직접 변경 (선택사항)

기존 변수 대신 표준 변수로 직접 변경:
- `var(--fs-xs)` → `var(--fs-micro)`
- `var(--fs-md)` → `var(--fs-base)`
- `var(--fs-lg)` → `var(--fs-xl)`

## CTO 관점 평가

### 확장성 ✅
- 변수 개수 감소로 선택 어려움 해소
- 명확한 구조로 새로운 크기 추가 용이

### 일관성 ✅
- 동일 크기는 동일 변수 사용
- 중복 제거로 일관성 확보

### 유지보수성 ✅
- 변수 개수 감소로 관리 용이
- 명확한 구조로 이해하기 쉬움
- 레거시 호환성 유지
