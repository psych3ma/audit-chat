# 디자인 QA — 회계법인 제출용 개발 포트폴리오

회계법인 제출용이라는 전제에서 **디자이너 관점**으로 리포트 UI를 샤프하게 검토한 결과입니다. “보고서가 믿을 만하게 보이는가”, “정보 위계와 톤이 제출용에 맞는가”를 기준으로 했습니다.

---

## 1. 잘 되어 있는 점

| 항목 | 평가 |
|------|------|
| **2단 구조** | 왼쪽 관계도·오른쪽 텍스트 구성이 정리되어 있고, 스캔하기 좋음. |
| **상태 색상** | 수임 불가(빨강)·안전장치 시 수임(주황)·수임 가능(녹색)이 직관적. |
| **섹션 구분** | 주요 이슈(주황계)·권고 안전장치(녹색계)·근거 법령(회색)으로 역할이 구분됨. |
| **근거 법령 링크** | 파란색+밑줄+외부 링크 아이콘으로 클릭 가능성이 명확함. |
| **이모지 제거** | 섹션 타이틀에 이모지 없음 → 제출용 톤에 맞음. |

---

## 2. 반드시 손보면 좋은 부분 (제출용 관점)

### 2.1 정보 위계·가독성

| 이슈 | 현재 | 권장 |
|------|------|------|
| **헤더 상태와 검토 ID 비중** | 상태("수임 불가")와 검토 ID가 같은 줄에 있어, ID가 시선을 뺄 수 있음. | 검토 ID는 폰트 크기·명도 확실히 낮추고(예: 10px, opacity 0.85), “검토 ID:” 라벨과 함께 보조 정보로만 보이게. |
| **섹션 타이틀 크기** | 11px, uppercase. 제출용 인쇄 시 작게 느껴질 수 있음. | 11px 유지해도 되나, **굵기(700)** 와 **자간(letter-spacing)** 은 유지. 인쇄/PDF 제출을 전제로 한다면 12px 검토. |
| **검토 의견 본문** | 12px, line-height 1.55. | 본문은 12px 유지하되, **line-height 1.6~1.65** 로 조금만 늘리면 제출용 가독성에 유리. |

### 2.2 색상·스타일 일관성

| 이슈 | 현재 | 권장 |
|------|------|------|
| **주요 이슈 박스 배경** | `.independence-report .issue-box` 가 `#fff3e0`(주황빛). 일반 `.issue-box` 는 `#fff5f3`(붉은빛). | 독립성 리포트만 쓰는 경우 한 가지로 통일. 제출용이라면 **#fff5f3 + 좌측 red 보더** 조합이 “이슈” 연상에 맞고, 주황(#fff3e0)은 “경고/조건부”에 두는 쪽이 의미 일관됨. |
| **오른쪽 본문 배경** | `#f8f9fa`. 왼쪽 viz-wrap `#fafafa` 와 비슷해 차이가 작음. | 구분을 유지하려면 `#f8f9fa` 유지. “전부 흰색”으로 통일할 경우 인쇄 시 더 단정해 보일 수 있음 → 제출 채널에 따라 선택. |
| **법령 링크 색** | `#1565c0`. | 접근성(대비)·일관성 위해 `:hover` 시 밑줄 유지하고, 필요 시 `color` 를 한 단계 더 진하게 검토(예: #0d47a1). |

### 2.3 관계도(시각화) 영역

| 이슈 | 현재 | 권장 |
|------|------|------|
| **빈 상태** | Mermaid 미생성 시 viz-wrap만 비어 있음. | “관계도를 생성할 수 없습니다” 같은 짧은 문구를 넣어, 오류/미생성 구분이 되게 하면 제출용 설명에 유리. |
| **이미지 비율** | `max-width: 100%` 로만 제한. | 세로가 과하게 길어지지 않도록 `max-height: 280px` 등 상한을 두고 object-fit으로 비율 유지하면, 리포트 전체 균형이 좋아짐. |

### 2.4 타이포·여백

| 이슈 | 현재 | 권장 |
|------|------|------|
| **리스트 불릿** | `padding-left: 16px`, `::before` 로 "—" 사용. | 제출용에서는 **실제 bullet(•) 또는 번호** 가 더 익숙할 수 있음. "—" 유지 시에는 시각적 굵기·색이 본문과 구분되게. |
| **권고 안전장치 없음** | “권고 안전장치 없음” 한 줄만 나올 수 있음. | 문맥에 따라 “해당 없음” 등으로 통일하거나, 회색 작은 글씨로 “이 건의 경우 권고 안전장치가 없습니다”처럼 한 문장으로 주면 제출용 톤에 맞음. |

## 3. 제출용 포트폴리오에서 보강하면 좋은 부분

### 3.1 인쇄·PDF 대응

- **인쇄 시** 배경색이 빠지거나 관계도가 잘리는 경우가 있을 수 있음.
- `@media print` 에서  
  - 카드 배경·박스 배경을 `print-color-adjust: exact` 또는 `-webkit-print-color-adjust: exact` 로 유지하고,  
  - 관계도 영역에 `break-inside: avoid` 를 주면, 페이지 나눔 시 관계도가 중간에 잘리지 않게 할 수 있음.

### 3.2 접근성(포트폴리오 플러스)

- 헤더 상태(“수임 불가” 등)에 `aria-live="polite"` 또는 역할(role)을 두면, 스크린 리더 사용자도 결과 요약을 먼저 인지할 수 있음.
- 근거 법령 링크에 `rel="noopener noreferrer"` 와 함께 **링크 목적**을 설명하는 `aria-label`(예: “공인회계사법 제21조 조문 보기”)을 넣으면 제출용 문서에서 접근성 의도를 보여주기 좋음.

### 3.3 톤 일관성

- 모든 사용자 대면 문구(버튼, 빈 상태, 에러 메시지, “권고 안전장치 없음” 등)가 **한국어·존댓말·동일한 어휘 수준**으로 되어 있으면, 회계법인 제출용 포트폴리오에서 “완성도”로 어필하기 좋음. (현재도 대체로 일관됨.)

## 4. QA 체크리스트 (제출 전 한 번만)

- [ ] 헤더에서 상태가 가장 크고, 검토 ID는 보조적으로만 보이는가?
- [ ] 주요 이슈·권고 안전장치 박스 색이 서로 구분되고, 독립성 카드 내에서 한 가지 체계로 통일되어 있는가?
- [ ] 관계도가 없을 때 빈 칸만 있지 않고, 짧은 설명 문구가 있는가?
- [ ] 근거 법령이 링크일 때 hover 시 밑줄·색 변화가 명확한가?
- [ ] 인쇄(또는 PDF 저장) 시 배경색·관계도 잘림이 없는가?
- [ ] 전체 문구가 한국어·동일 톤으로 통일되어 있는가?

## 5. 요약

- **레이아웃·색상 역할·이모지 제거**는 이미 제출용에 맞게 잘 잡혀 있음.
- **정보 위계(헤더 vs 검토 ID)·색상 통일(이슈 박스)·관계도 빈 상태·인쇄 대응**을 보완하면, “회계법인에 제출하는 개발 포트폴리오”로서 디자인 완성도와 신뢰도가 더 올라갑니다.
- 위 권장 사항은 우선순위대로 적용한 뒤, 실제 인쇄/PDF로 한 번 출력해 보면서 조정하는 것을 권장합니다.
