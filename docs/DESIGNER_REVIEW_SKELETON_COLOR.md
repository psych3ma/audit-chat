# 스켈레톤 UI 색상 및 테두리 검토 - 디자이너 관점

**작성일**: 2026-02-12  
**작성자**: 디자이너 전문가 관점  
**검토 대상**: 스켈레톤 UI의 색상 강도 및 테두리 디자인

---

## 현재 문제 분석

### 발견된 문제점

1. **색상이 너무 강함**
   - 빨간색 계열(`rgba(173, 27, 2, ...)`) shimmer 효과가 눈에 띄게 강함
   - 중성적인 배경과 대비되어 시선을 과도하게 끔
   - 로딩 상태임을 알리기보다는 "강조된 요소"처럼 보임

2. **테두리가 눈에 띄게 구분됨**
   - `1.5px solid var(--border)` 테두리가 각 칩을 명확히 구분
   - "박스" 느낌이 강해 통합되지 않은 느낌
   - 미니멀한 디자인 철학과 맞지 않음

3. **전체적인 시각적 균형**
   - 스켈레톤 UI가 실제 콘텐츠보다 더 눈에 띄는 역효과
   - 로딩 상태는 "배경"처럼 느껴져야 하는데 "전경"처럼 보임

---

## 디자인 원칙 분석

### 1. 스켈레톤 UI의 역할

**핵심 가치**:
- 로딩 중임을 **미묘하게** 전달
- 실제 콘텐츠가 곧 나타날 것을 암시
- **방해하지 않으면서** 기대감 유지

**현재 문제**:
- 너무 눈에 띄어서 시선을 끔
- 로딩 상태가 주인공처럼 보임

### 2. 미니멀리즘 원칙

**원칙**:
- 불필요한 시각적 요소 제거
- 색상은 중성적이고 미묘하게
- 테두리는 최소화하거나 제거

**현재 부족한 점**:
- 빨간색 계열이 너무 강함
- 테두리가 명확히 구분됨

### 3. 색상 선택의 원칙

**스켈레톤 UI에 적합한 색상**:
- 중성적인 회색 계열
- 배경과 자연스럽게 어우러짐
- 브랜드 색상은 실제 콘텐츠에만 사용

**현재 문제**:
- 브랜드 색상(빨간색)을 스켈레톤에 사용
- 중성적이지 않아 눈에 띔

## 개선 방안

### 옵션 A: 완전 중성화 (권장)

**원칙**: 중성적인 회색 계열로 완전히 변경, 테두리 제거

#### 1. 색상 변경
- Shimmer 효과: 빨간색 → 중성 회색 계열
- 내부 플레이스홀더: 빨간색 → 중성 회색 계열
- opacity는 유지하여 미묘함 보장

#### 2. 테두리 제거
- `border` 완전 제거
- 배경색만으로 구분
- 더 통합된 느낌

**장점**:
- 미니멀하고 통합된 느낌
- 눈에 띄지 않으면서도 로딩 상태 전달
- 실제 콘텐츠가 나타날 때 자연스러운 전환

**단점**:
- 브랜드 색상과의 연결감 약화 (하지만 스켈레톤이므로 허용 가능)

---

### 옵션 B: 미묘한 브랜드 힌트 유지

**원칙**: 빨간색을 매우 미묘하게 유지, 테두리만 제거

- Shimmer: 빨간색 opacity를 더 낮춤 (0.03-0.04)
- 테두리: 제거
- 내부 플레이스홀더: 중성 회색으로 변경

**장점**:
- 브랜드 연결감 유지
- 여전히 미묘함

**단점**:
- 여전히 눈에 띌 수 있음

---

## 디자이너 권장사항

### 최종 권장안: 옵션 A (완전 중성화)

**이유**:

1. **스켈레톤 UI의 본질**
   - 스켈레톤은 "임시 상태"이므로 브랜드 색상이 필요 없음
   - 실제 콘텐츠가 나타날 때 브랜드 색상이 적용됨

2. **시각적 균형**
   - 중성적인 색상으로 배경처럼 느껴짐
   - 실제 콘텐츠가 나타날 때 더 돋보임

3. **미니멀리즘**
   - 테두리 제거로 더 통합된 느낌
   - 불필요한 시각적 요소 제거

---

## 구현 가이드라인

### 1. 색상 변경 (중성 회색 계열)

```css
/* Shimmer 효과 색상: 중성 회색 계열 */
--skeleton-shimmer-color: rgba(0, 0, 0, 0.03);
--skeleton-shimmer-color-strong: rgba(0, 0, 0, 0.05);
--skeleton-shimmer-label-bg: rgba(0, 0, 0, 0.06);
--skeleton-shimmer-text-bg: rgba(0, 0, 0, 0.04);
```

### 2. 테두리 제거

```css
.chip-skeleton {
  /* border: 1.5px solid var(--border); 제거 */
  border: none;
  /* 또는 완전히 제거 */
}
```

### 3. 전체적인 미묘함 강화

- Shimmer 효과의 opacity를 더 낮춤
- 내부 플레이스홀더도 더 미묘하게
- 배경과 자연스럽게 어우러지도록

---

## 사용자 경험 개선 효과

### Before (현재)
- 빨간색 shimmer가 눈에 띔
- 테두리로 명확히 구분됨
- 스켈레톤이 "전경"처럼 보임
- 실제 콘텐츠보다 더 눈에 띄는 역효과

### After (개선 후)
- 중성적인 회색 shimmer로 미묘함
- 테두리 제거로 통합된 느낌
- 스켈레톤이 "배경"처럼 느껴짐
- 실제 콘텐츠가 나타날 때 자연스러운 전환

---

## 디자인 원칙 준수

1. **미니멀리즘**: 불필요한 시각적 요소 제거
2. **균형**: 스켈레톤은 배경처럼, 실제 콘텐츠는 전경처럼
3. **일관성**: 중성적인 색상으로 통일
4. **명확성**: 여전히 로딩 상태임을 전달하되, 방해하지 않음

---

## 결론

**권장 접근 방식**:
- ✅ Shimmer 효과: 빨간색 → 중성 회색 계열
- ✅ 테두리 제거: `border: none`
- ✅ 내부 플레이스홀더: 중성 회색 계열
- ✅ 전체적인 미묘함 강화

이 접근 방식은 **미니멀하고 통합된 느낌**을 제공하며, 스켈레톤 UI가 눈에 띄지 않으면서도 로딩 상태를 전달합니다.
