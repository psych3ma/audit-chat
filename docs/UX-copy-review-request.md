# UX Copy 수정 요청서

**작성일:** 2025-02-12  
**대상:** UX Writer  
**참조:** 회계감사 전문가 검토 + UX Writer 1차 검토  
**적용 화면:** `static/audit-chat-pwc.html` (PwC 스타일 감사 독립성 검토 UI)

---

## 1. 배경 및 목적

본 서비스는 **감사인 독립성(Audit Independence)** 시나리오를 입력받아 수임 가능성·주요 이슈·권고 안전장치를 분석하는 AI 검토 도구입니다.  
회계감사 전문가 관점에서 **용어 정확성**, **규정·실무와의 정합성**, **사용자(감사·품질관리 담당자) 기대에 맞는 문구**를 확보하기 위해 아래 수정을 요청드립니다.

---

## 2. 회계감사 전문가 관점 검토 요약

### 2.1 서비스 정체성 불일치 (Critical)

| 구분 | 현재 문구 | 이슈 |
|------|-----------|------|
| 서비스 성격 | 화면 전반에 「내부통제」 | 실제 시나리오·API·리포트는 **감사 독립성** 전용입니다. 「내부통제」는 별도 개념(내부통제기준, COSO 등)이라 혼동을 줍니다. |
| 시나리오 내용 | 퇴직 후 임원 취임, 전 경리과장 입사, 직계비속 주식 보유 등 | 전부 **감사인 독립성 규정**(예: KICPA 독립성 준칙, 지배구조) 관련 사례입니다. |

**요청:** 서비스명·타이틀·설명을 **「감사 독립성」** 또는 **「감사인 독립성 검토」**에 맞춰 수정해 주세요.

### 2.2 결과물 설명 불일치

| 위치 | 현재 문구 | 이슈 |
|------|-----------|------|
| empty-sub | 「통제 적절성, 관련 기준, 개선 방향을 분석해드립니다」 | 내부통제 검토 결과에 가까운 표현입니다. 감사 독립성 검토 결과는 **수임 가능성, 독립성 위험(주요 이슈), 권고 안전장치** 등입니다. |

**요청:** 분석 결과로 제공되는 내용(수임 가능/불가, 주요 이슈, 권고 안전장치, 검토 의견)을 사용자에게 정확히 전달하는 문구로 바꿔 주세요.

### 2.3 전문 용어·일관성

| 항목 | 현재 | 검토 의견 |
|------|------|-----------|
| 리포트 헤더 | 「위험도」 | 감사 독립성 맥락에서는 「독립성 위험 수준」 등이 더 적절할 수 있습니다. (유지 시에는 일관 사용) |
| 섹션명 | 「주요 이슈」 / 「권고 안전장치」 | 실무에서 통용되는 표현으로 적절합니다. 유지 권장. |
| 폴백 메시지 | 「로컬 시나리오로 표시합니다」 | “로컬”이 기술 용어로 들려, 사용자에게는 **「연결 실패 시 예시 기준 결과를 표시합니다」** 등이 명확합니다. |
| 리포트 구분 | 독립성 카드: 「검토 의견」 / 폴백 카드: 「품질관리실 검토 의견」 | 두 리포트 타입에서 섹션명을 통일할지 결정 후, 동일 용어 적용 요청. |

## 3. UX Writer 수정 요청 목록

아래는 **수정 요청 항목**, **현재 문구**, **수정 제안(회계감사 관점 반영)** 을 정리한 표입니다. 제안은 참고용이며, 톤·길이·브랜드 가이드에 맞게 조정 가능합니다.

### 3.1 서비스명·타이틀·Empty State (우선순위: 높음)

| No | 위치 | 현재 문구 | 수정 요청 / 제안 |
|----|------|-----------|------------------|
| 1 | 페이지 타이틀 (`<title>`) | 삼일PwC — 내부통제 AI 검토 | 서비스가 **감사 독립성** 검토이므로, 예: **「삼일PwC — 감사 독립성 AI 검토」** 등으로 변경 요청. |
| 2 | 헤더 서비스명 | 내부통제 검토 AI | **「감사 독립성 검토 AI」** 또는 **「감사인 독립성 AI 검토」** 등으로 변경 요청. |
| 3 | empty-eyebrow | Internal Control Review | 한글 통일 시 **「감사 독립성 검토」**, 영문 유지 시 **「Audit Independence Review」** 로 변경 요청. |
| 4 | empty-title | 내부통제 시나리오를 AI로 검토해보세요 | **「감사 독립성 시나리오를 AI로 검토해보세요」** 로 변경 요청. |
| 5 | empty-sub | 시나리오를 선택하면 통제 적절성, 관련 기준, 개선 방향을 분석해드립니다. | 감사 독립성 결과물에 맞게, 예: **「시나리오를 선택하면 수임 가능성, 주요 이슈, 권고 안전장치를 검토해드립니다.»** 등으로 변경 요청. |

### 3.2 입력 영역·안내 문구 (우선순위: 높음)

| No | 위치 | 현재 문구 | 수정 요청 / 제안 |
|----|------|-----------|------------------|
| 6 | textarea placeholder | 위 예시 버튼을 선택하면 시나리오가 입력됩니다 | • 「위」가 스크롤 시 애매할 수 있음. • 「예시 버튼」과 다른 문구의 「시나리오 버튼」 통일. 제안: **「시나리오를 선택하면 여기에 입력됩니다」** 또는 **「아래 시나리오 중 하나를 선택하세요」** 등. |
| 7 | input-hint | 위 시나리오 버튼을 눌러 분석을 시작하세요 | • 「검토」와 「분석」 용어 통일(본 서비스는 검토 성격이므로 「검토」 권장). • 「눌러」→「선택하면」 등 톤 통일. 제안: **「시나리오를 선택한 뒤 전송하면 검토를 시작합니다」** 등. |

### 3.3 칩·CTA (우선순위: 중간)

| No | 위치 | 현재 문구 | 수정 요청 / 제안 |
|----|------|-----------|------------------|
| 8 | chip-arrow (초기 칩) | 이 시나리오로 검토하기 → | 유지 가능. 전역에서 「검토」로 통일 시 일관됨. |
| 9 | post-chips-label (답변 후) | 다른 시나리오를 선택하세요 | 적절함. 필요 시 **「다른 시나리오로 검토하기」** 등 CTA 톤과만 통일. |

### 3.4 버튼·시스템 메시지 (우선순위: 높음)

| No | 위치 | 현재 문구 | 수정 요청 / 제안 |
|----|------|-----------|------------------|
| 10 | 전송 버튼 | (아이콘만 있음) | 접근성: **aria-label="검토 요청"** 또는 **"전송"** 등 스크린리더용 라벨 추가 요청. |
| 11 | reset-btn | ↩ 새 분석 | 「새 분석」이 “대화 초기화”인지 “새 시나리오 검토”인지 명확히. 제안: **「↩ 처음부터 다시」** 또는 **「새 검토 시작」** 등. |
| 12 | Beta 뱃지 | Beta | (선택) 툴팁 등으로 **「실험 단계 서비스입니다」** 등 짧은 설명 추가 요청. |
| 13 | API 실패 시 에러 메시지 | 독립성 검토 API 연결 실패. 로컬 시나리오로 표시합니다. | **「검토 요청 연결에 실패했습니다. 예시 기준 결과를 표시합니다.»** 등 사용자 관점·기술 용어 최소화로 수정 요청. (에러 상세는 별도 영역에만 표시 권장) |

### 3.5 리포트 카드 내 문구 (우선순위: 중간)

| No | 위치 | 현재 문구 | 수정 요청 / 제안 |
|----|------|-----------|------------------|
| 14 | 독립성 리포트 – 이슈 없을 때 | 위반 사항 없음 | 맥락에 맞게 **「해당 없음」** 또는 **「주요 이슈 없음」** 등으로 통일 검토. |
| 15 | 독립성 리포트 – 안전장치 없을 때 | 추가 조치 필요 없음 | **「권고 안전장치 없음」** 또는 **「해당 없음」** 등 섹션명과 맞게 통일 검토. |
| 16 | 리포트 ID 표기 | ID: {traceId} / REVIEW NO. {id} | 전문가용이면 유지. 한글 라벨 필요 시 **「검토 ID」**, **「검토 번호」** 등 보완 요청. |
| 17 | 적용 기준 라벨 | 적용 기준 : | 띄어쓰기 일관성(**적용 기준:** 등) 및 다른 라벨과 형식 통일 요청. |
| 18 | 폴백 카드 섹션명 | 품질관리실 검토 의견 | 독립성 카드의 「검토 의견」과 통일할지 결정 후, **「검토 의견」** 으로 통일하거나, 폴백만의 맥락을 살린 **「검토 의견 (예시 기준)」** 등 검토 요청. |
| 19 | 관계도 이미지 alt | 관계도 | **「감사·회사 간 관계도」** 등 보다 설명적인 alt 문구로 수정 요청(접근성). |

### 3.6 용어 통일 체크리스트 (전체)

아래 용어를 화면 전반에서 동일한 표현으로 사용해 주세요.

| 용어 | 통일 제안 | 비고 |
|------|-----------|------|
| 서비스 영역 | **감사 독립성** (또는 감사인 독립성) | 「내부통제」는 본 화면에서 사용하지 않음. |
| 사용자 행동 | **검토** | 「분석」과 혼용 지양. |
| UI 요소 | **시나리오** | 「예시 버튼」「시나리오 버튼」 → 「시나리오」로 통일. |
| 리포트 섹션 | **관계도** vs **관계 지형도** | 한 가지로 통일(예: **관계도**). |
| 이모지 | 🗺️/🗺, 🚨/⚠, 🛡️/🛡 | 동일 의미면 하나로 통일 권장. |

## 4. 수정 시 참고 사항

- **대상 사용자:** 회계법인·감사·품질관리 담당자. 전문 용어(수임 가능, 안전장치, 감사대상회사 등) 사용 가능.
- **규정·실무:** 감사인 독립성 관련 KICPA 준칙·지배구조 논의와 혼동이 없도록, 「내부통제」와 「감사 독립성」을 구분해 주시면 됩니다.
- **접근성:** 버튼·칩에 스크린리더용 라벨(또는 aria-label) 부여 요청.
- **에러·폴백:** 기술 용어(로컬, API 등)는 최소화하고, “무엇이 안 됐고, 지금 무엇이 보이는지”를 사용자 관점으로 서술해 주세요.
