# 프로그레스 인디케이터 — 디자인 핸드오프

> **대상**: 개발  
> **참고**: `docs/WORKFLOW_3STEPS.md` (진행 단계·문구 정의)  
> **목적**: 로딩 중 표시할 3단계 프로그레스 UI 스펙 전달

---

## 1. 레이아웃 구조

진행 상태는 **3행**으로 구성한다.

| 행 | 내용 | 정렬 | 비고 |
|----|------|------|------|
| **1행** | 기존 상태바 (점 3개) | **가운데** | 현재 단계에 해당하는 점 강조, 나머지 연회색 |
| **2행** | (n/3) + 단계명 | **왼쪽** | 예: `(1/3) 관계 추출` |
| **3행** | 단계 설명 문구 | **왼쪽** | 예: `시나리오에서 인물·회사·관계 구조를 추출하고 있습니다.` |

- 1행만 가운데 정렬, 2·3행은 왼쪽 정렬로 텍스트 가독성·정보 위계 유지.
- 전체 블록은 기존 대화 버블/카드와 동일한 컨테이너 안에 배치.

---

## 2. 문구 (WORKFLOW_3STEPS 기준)

단계별 **2행·3행** 문구는 아래를 사용한다. (문구 출처: `docs/WORKFLOW_3STEPS.md`)

| 단계 | 2행 표시 | 3행 표시 |
|------|----------|----------|
| 1/3 | (1/3) 관계 추출 | 시나리오에서 인물·회사·관계 구조를 추출하고 있습니다. |
| 2/3 | (2/3) 독립성 분석 | 추출된 구조를 바탕으로 수임 가능성과 위험을 판단하고 있습니다. |
| 3/3 | (3/3) 보고서 생성 | 결론, 관계도, 법령 링크를 정리해 보고서를 만들고 있습니다. |

- 2행: `(n/3)` + 공백 한 칸 + **단계명** (위 표의 “2행 표시” 컬럼).
- 3행: **설명 문구만** (위 표의 “3행 표시” 컬럼). 중간점(·) 유지.

---

## 3. 시각·UX 요구사항 (디자이너 관점)

| 항목 | 요구사항 |
|------|----------|
| **상태바 (1행)** | 점 3개, 가운데 정렬. 현재 단계 점은 브랜드 포인트 색(예: 오렌지), 나머지는 연한 회색. 진행감이 있으면 pulse 등 소폭 애니메이션 가능. |
| **2·3행 텍스트** | 검정보다는 **눈에 보이는 회색** 톤 권장. 단계명(2행)은 굵게, 설명(3행)은 보통 굵기. |
| **정렬** | 1행: `text-align: center` / 2·3행: `text-align: left`. 컨테이너는 기존 최대 너비 유지. |
| **접근성** | 단계 전환 시 스크린리더 안내용(예: `aria-live`, `aria-label`) 유지 권장. `prefers-reduced-motion` 시 애니메이션 제거. |

---

## 4. 개발 체크리스트

- [ ] 1행: 기존 상태바(점 3개) 유지, **가운데 정렬** 적용
- [ ] 2행: `(n/3) 단계명` 한 줄, **왼쪽 정렬**, 단계 변경 시 문구만 교체
- [ ] 3행: 설명 문구만, **왼쪽 정렬**, 위 표 3행 문구 사용
- [ ] 문구 데이터: `WORKFLOW_3STEPS.md` 또는 동일 내용의 단일 소스(WORKFLOW_STEPS 등) 참고
- [ ] 2·3행 색상: 회색 계열로 통일
- [ ] (선택) 접근성·reduced-motion 반영
