## ⏱️ 시간 비용 분석

### Phase 1 작업 분해

#### 1. HTML 파싱 모듈 개발

**작업 내용**:
- BeautifulSoup4 기반 HTML 파서 클래스 작성
- 법령 메타데이터 추출 (법령명, 시행일, 개정일)
- 계층 구조 파싱 (장 → 조문 → 항목 → 호목)
- 텍스트 정규화 (HTML 태그 제거, 공백 정규화)
- 에러 핸들링 및 검증 로직

**예상 코드량**: 200-300줄

**시간 추정**:
- 기본 파싱 로직: **2-3시간**
- 계층 구조 추출: **2-3시간**
- 텍스트 정규화: **1시간**
- 에러 핸들링: **1-2시간**
- **소계**: **6-9시간**

**난이도**: ⭐⭐⭐ (중상)
- HTML 구조 일관성 의존
- 계층 구조 파싱 복잡도 중간

#### 2. JSON 구조 설계

**작업 내용**:
- JSON 스키마 설계
- Pydantic 모델 정의 (선택적)
- 구조 검증 로직

**예상 코드량**: 50-100줄

**시간 추정**:
- 스키마 설계: **1시간**
- 모델 정의: **1시간**
- **소계**: **2시간**

**난이도**: ⭐ (낮음)

#### 3. 파싱 실행 및 검증

**작업 내용**:
- 6개 HTML 파일 일괄 파싱
- 파싱 결과 검증 (조문 수, 구조 일관성)
- 수동 검토 (핵심 조문)

**시간 추정**:
- 파싱 실행: **0.5시간** (자동화)
- 결과 검증: **1-2시간**
- 수동 검토: **1-2시간**
- **소계**: **2.5-4.5시간**

**난이도**: ⭐⭐ (중간)

#### 4. 문서화 및 테스트

**작업 내용**:
- 코드 주석 및 문서화
- 단위 테스트 작성 (선택적)
- 사용 가이드 작성

**시간 추정**:
- 문서화: **1시간**
- 테스트: **1-2시간** (선택적)
- **소계**: **1-3시간**

**난이도**: ⭐⭐ (중간)

## 💾 저장 용량 분석

### JSON 파일 크기 예상

#### HTML → JSON 변환 비율

**HTML 구조 분석**:
- HTML 태그 및 스타일: ~30-40% (불필요한 정보)
- 실제 텍스트 내용: ~60-70%

**JSON 구조 오버헤드**:
- JSON 키 이름: ~10-15%
- 구조화 메타데이터: ~5-10%

**예상 변환 비율**: HTML 1MB → JSON **0.5-0.7MB** (50-70%)

#### 파일별 예상 크기

| HTML 파일 | HTML 크기 | 예상 JSON 크기 |
|-----------|-----------|----------------|
| 공인회계사법(법령단위비교) | 222KB | **110-155KB** |
| 공인회계사법(위임조문) | 239KB | **120-167KB** |
| 공인회계사법(인용조문) | 256KB | **128-179KB** |
| 외부감사법(법령단위비교) | 320KB | **160-224KB** |
| 외부감사법(위임조문) | 281KB | **141-197KB** |
| 외부감사법(인용조문) | 338KB | **169-237KB** |

**총 예상 JSON 크기**: **828KB - 1.16MB**

**CTO 권장 추정**: **~1MB** (보수적)

#### 압축 고려사항

**gzip 압축**:
- JSON 텍스트 압축률: ~70-80%
- 압축 후 크기: **~200-300KB**

**권장사항**:
- 저장: 압축 없이 JSON (가독성)
- 전송: 필요 시 gzip 압축

### Neo4j 저장 용량 (Phase 2, 참고)

**Neo4j 저장 오버헤드**:
- 노드: ~100-200 bytes/노드
- 관계: ~50-100 bytes/관계
- 속성: 텍스트 크기 + 오버헤드

**예상 노드 수**:
- 법령: 2개
- 장: ~10-15개
- 조문: ~100-150개
- 항목/호목: ~500-1000개

**예상 Neo4j 용량**: **~500KB - 1MB**
